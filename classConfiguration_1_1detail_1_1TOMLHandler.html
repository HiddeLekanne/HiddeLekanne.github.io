Detailed Description

Stores parsed data of a TOML file in a std::map<string, TOMLValue>.

This class implements the interface required by the loltoml parser. Each function is called on a parsing event. It is required to deliver an implementation of each of these functions. Not handling that event equals providing an empty implementation.

Warning
The handler has only implemented a part of the TOML v0.4.0 specification. The following sections of the TOML v0.4.0 specification are handled:
  • Comment
  • String
  • Integer
  • Float
  • Boolean
  • Array
  • Table
The following sections of the TOML v0.4.0 specification have not yet been fully implemented in the handler:
  • Inline Table: no support for inline tables in arrays.
The following sections of the TOML v0.4.0 specification have not yet been implemented in the handler:
  • Datetime, including "Offset Date-Time", "Local Date-Time", "Local Date" and "Local Time" - see here
  • Array of Tables

This class is used by parseConfig for loltoml::parse.

◆ current_table_

std::vector<std::string> Configuration::detail::TOMLHandler::current_table_
private

Stores the current table.

If empty, no table is set. Multiple items are joined by ".", as specified by the TOML standard.

This "split-up" version of the full table name is stored, so it is easier to navigate up in the table tree.

◆ current_table_string_

A "flat" string representation of the current table.

This equals StringTools::join(current_table_, ".").

Storing this separately makes it easier to compare keys.
