Detailed Description

This module finds balls by processing ROI's, a ROI is a square which is the size of a ball at a certain spot in the image.

The first check is if a ROI contains enough bright pixels to contain a ball. If this is the case, a second check is applied, is there a blob in the image which is about the right size to be a blob on the ball. If this is also the case, the ROI is put into a haar classifier which decides whether a ROI is a ball or not.

◆ getNextROI()

bool mBallDetector::getNextROI ( ROI &  current_roi)
private

Based on the current ROI, find the next ROI that needs processing.

Either move sideways, or upwards if an entire row has been processed. A Roi's size is based on the expected ball size on that position in the image supplied by the computeExpectedBallSize function. Whether it's a ROI or not is based on whether the amount of white pixels is in between MIN_PERCENTAGE_NON_ZEROS_ and MAX_PERCENTAGE_NON_ZEROS_.

Parameters
current_roiThe roi we are currently checking.
Returns
The next roi.
