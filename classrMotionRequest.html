Performs an in-walk kick.

Speed should be set as well to ensure the robot keeps walking after kicking.

The values listed here are suggestions, they were tested on a few robots and worked. They can be used as a starting point for making a kick in specific situations, or used as-is if they satisfy the goal.

Parameters
xForward speed in mm/s.
yLeftward speed in mm/s.
rotationCounter clockwise rotation in radian/s.
kick_leftIf true the left foot will be the kicking foot, otherwise the right foot.
kick_typeKickType: how strong the kick should be

◆ walk_mode_

rWalkGenerator::WalkMode rMotionRequest::walk_mode_

walkMode sets the type of walk to be used.

This can be either one of: rWalkGenerator::WalkMode::speedMode - Speed in mm/s and radians/s. rWalkGenerator::WalkMode::stepSizeMode - Step size in mm and radians (robot will do exact steps). rWalkGenerator::WalkMode::targetMode - UNDER DEVELOPMENT. Walk to given relative position in mm and radians. rWalkGenerator::WalkMode::kickMode - Perform an in-walk kick using the walking engine. The user should set a speed to ensure that the robot walks after kick, if no speed is set, the previous speed is used.

◆ walk_speed_

Pose2f rMotionRequest::walk_speed_ = Pose2f()

Set walk_speed_, step_size_ or target_ with: Pose2f(rotation, Eigen::Vector2f(forward, sideways)).

Maximum values for rotation, forward and sideways can be set or found in the walking engine. The max speed for the robot to walk with, used during speedMode and targetMode.
