DNT2017

Tested working requirements version numbers:

  • clang 3.8
  • cmake 3.5.1
  • Ubuntu 16.04.2 LTS
  • doxygen 1.8.11
  • opencv-3.3
  • qt 5.5.1
  • libjpeg-turbo8-dev

Before you try to build or copy to robots, first install some system-wide settings:

cd util/scripts
./install_dnt

Before you copy a new executable to a fresh robot, first install our framework on the robot using:

cd util/scripts
./install_robot <ip-address>

To copy the new executable to a robot, call:

robocopy <ip-address>

To view the possible command line argument for configuring the robot, call:

robocopy --help

To compile the documentation using doxygen, from the root project directory, call:

doxygen

After completion, the html or latex subfolders in docs will contain the documentation files. A useful start point is the docs/html/index.html file.

Ubuntu-18.04:

Only strictly necessary downgrade is to clang 3.8 (how-to below): Other programs have most likely a equal to or greater than version dependency, a tested build is:

  • cmake 3.10.2 (newest atm) sudo apt install cmake
  • doxygen 1.8.13 (newest atm) sudo apt install doxygen
  • opencv-3.4.3 (easily accessable atm)
  • qt 5.9.5 (newest atm) sudo apt install qt5-default

To add the repository, install the key and install clang 3.8. Also clang 6 (or newer) is needed to read your own gcc libraries. For clang installations you need to enable universe and multiverse in your "software and updates" menu:

sudo apt-get install clang
sudo apt-add-repository "deb http://apt.llvm.org/xenial/ llvm-toolchain-xenial-3.8 main"
wget -O - https://apt.llvm.org/llvm-snapshot.gpg.key | sudo apt-key add
sudo apt update
sudo apt-get install clang-3.8
